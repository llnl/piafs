Quick Start
===========

This page provides a quick overview of the PIAFS workflow.

PIAFS Workflow
--------------

Running a PIAFS simulation involves these steps:

1. **Prepare input files** - Define the simulation parameters
2. **Generate initial solution** - Create the initial condition
3. **Run the solver** - Execute PIAFS
4. **Visualize results** - Plot and analyze the output

Input Files
-----------

PIAFS requires several input files:

Required Files
~~~~~~~~~~~~~~

* ``solver.inp`` - Main solver parameters (grid size, time stepping, schemes)
* ``boundary.inp`` - Boundary conditions
* ``physics.inp`` - Physical model parameters (gamma, Reynolds number, etc.)
* ``initial.inp`` - Initial solution (generated by initialization code)

Optional Files
~~~~~~~~~~~~~~

* ``weno.inp`` - WENO scheme parameters
* ``lusolver.inp`` - LU solver parameters for compact schemes
* ``chemistry.inp`` - Photochemistry parameters

Typical Workflow
----------------

1. Choose an Example
~~~~~~~~~~~~~~~~~~~~

Start with one of the provided examples in the ``Examples/`` directory:

.. code-block:: bash

   cd Examples/1D_Euler/SodShockTube
   ls

You'll see input files (``solver.inp``, ``boundary.inp``, ``physics.inp``) and an ``aux/`` directory with initialization code.

2. Generate Initial Solution
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Compile and run the initialization code:

.. code-block:: bash

   cd aux
   gcc init.c -lm -o INIT
   cd ..
   ./aux/INIT

This creates ``initial.inp`` containing the initial solution.

3. Run PIAFS
~~~~~~~~~~~~

**Serial execution:**

.. code-block:: bash

   /path/to/piafs/bin/PIAFS-gcc-serial

**Parallel execution with MPI:**

.. code-block:: bash

   mpiexec -n 4 /path/to/piafs/bin/PIAFS-gcc-mpi

**On HPC systems with Slurm:**

.. code-block:: bash

   srun -n 16 /path/to/piafs/bin/PIAFS-gcc-mpi

PIAFS will display progress on the screen and write solution files at specified intervals.

4. Visualize Results
~~~~~~~~~~~~~~~~~~~~

Depending on the output format specified in ``solver.inp``:

**Text format** (``op_file_format = text``):

.. code-block:: bash

   gnuplot
   > plot 'op_00010.dat' using 1:2 with lines

**Binary format** (``op_file_format = binary``):

.. code-block:: bash

   export PIAFS_DIR=/path/to/piafs
   python $PIAFS_DIR/Examples/Python/plotSolution1D.py

**Tecplot format** (``op_file_format = tecplot2d``):

Open in Tecplot or VisIt.

Understanding Output
--------------------

PIAFS writes several types of output:

Solution Files
~~~~~~~~~~~~~~

* ``op_XXXXX.dat`` - Solution at time step XXXXX
* Format depends on ``op_file_format`` in ``solver.inp``

Screen Output
~~~~~~~~~~~~~

PIAFS displays:

* Startup information (version, configuration)
* Iteration number and simulation time
* Time step size (if using adaptive time stepping)
* Wall clock time

Log Files
~~~~~~~~~

* Conservation check results (if enabled)
* Error messages and warnings
