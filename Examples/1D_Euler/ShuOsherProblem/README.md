# Shu-Osher Problem

## Overview

A challenging test problem involving the interaction of a shock wave with an entropy wave. This case is commonly used to assess the resolution and shock-capturing capabilities of high-order numerical schemes.

## References

- C.-W. Shu and S. Osher, "Efficient implementation of essentially non-oscillatory schemes II," J. Comput. Phys., 83 (1989), pp. 32–78
- A. K. Henrick, T. D. Aslam, and J. M. Powers, "Mapped weighted essentially non-oscillatory schemes: Achieving optimal order near critical points," J. Comput. Phys., 207 (2005), pp. 542–567

## Governing Equations

1D Euler equations for inviscid, compressible flows:

```
d   [ rho   ]   d   [   rho*u    ]
--  [ rho*u ] + --  [rho*u*u + p ] = 0
dt  [   e   ]   dx  [ (e+p)*u    ]
```

Where:
- `rho` - density
- `u` - velocity
- `p` - pressure
- `e` - internal energy

Equation of state:
```
         p         1
e = -------  +   - rho * u^2
    gamma-1      2
```

## Domain and Boundary Conditions

- Domain: `-5.0 <= x <= 5.0`
- Boundary conditions: Extrapolate

## Initial Conditions

Shock region (`-5.0 <= x < -4.0`):
- `rho = 27.0/7.0`
- `u = 4.0*sqrt(35.0)/7.0`
- `p = 31.0/3.0`

Entropy wave region (`-4.0 <= x <= 5.0`):
- `rho = 1.0 + 0.2*sin(5x)`
- `u = 0`
- `p = 1`

## Required Input Files

- `solver.inp` - General solver parameters

See the main PIAFS README for other optional input files.

## How to Run

1. **Generate the initial solution:**
   ```bash
   cd Examples/1D_Euler/ShuOsherProblem
   g++ aux/init.C -o INIT
   ./INIT
   ```

2. **Run the simulation:**
   ```bash
   # Serial mode
   /path/to/piafs/bin/PIAFS-<compiler>-serial solver.inp

   # Parallel mode (e.g., 4 processes)
   mpiexec -n 4 /path/to/piafs/bin/PIAFS-<compiler>-mpi solver.inp
   ```

3. **Visualize results:**
   - If `op_file_format` is set to `text` in `solver.inp`, solution files can be plotted with Gnuplot or any plotting tool
   - Use Python scripts in `Examples/Python/` for binary format output

## Notes

- This problem produces complex wave interactions that challenge numerical schemes
- The code `aux/init.C` generates the initial solution file in the correct format required by the solver
